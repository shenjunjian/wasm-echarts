wasm-zrender 实例

wasm-zrender 文档

对齐官方 zrender export.ts 的 WASM 命名空间导出，底层封装 rust_zrender::ZRenderer。

快速开始

cd crates/wasm-zrender && wasm-pack build --target web

cd ../../site && npm install && npm run dev
# 打开 http://127.0.0.1:5173/zrender/examples/

顶层 API

  • init(dom?, opts?) — 创建 ZRender（dom 忽略，尺寸来自 opts）
  • registerFont(data, opts?) — 注册字体 bytes 到全局 fontdb（Text 渲染前必调，见下节）
  • clearFonts() — 清空已注册字体（测试用）
  • dispose(zr) / disposeAll() — 释放实例
  • getInstance(id) — 按 id 取实例

字体加载（Text 必看）

wasm-zrender 在 Rust 离屏 Canvas（vl-convert-canvas2d + cosmic-text）中绘制文本， 不经过浏览器 ctx.font。WASM / Wasmer 等宿主读不到操作系统字体， 因此 不会 在 .wasm 内嵌入默认字体，而由宿主在运行时注入。

使用 Text 图元或任何含文字的渲染前，须先注册至少一种字体；否则 refresh() 会报错 no default font found。

调用顺序

  1. await initWasm()
  2. registerFont(bytes, opts?) — 加载 TTF / OTF / WOFF 文件内容
  3. init(...) → 构图 → refresh()

可在任意时刻再次调用 registerFont；已创建的 ZRender 实例会自动热更新 fontdb。

registerFont(data, opts?)

  • data — Uint8Array，字体文件原始 bytes
  • opts.familyName — 可选，覆盖字体族名（不传则读字体 name table）
  • opts.sansSerif — 可选，string[]，将 CSS sans-serif 映射到这些族名；若只传 familyName 则默认映射到该族名
import initWasm, { init, registerFont, Text } from '@wasm-zrender/wasm_zrender.js';

await initWasm();

const bytes = new Uint8Array(
  await (await fetch('/fonts/NotoSansSC-Regular.ttf')).arrayBuffer()
);
registerFont(bytes, {
  familyName: 'Noto Sans SC',
  sansSerif: ['Noto Sans SC'],
});

const zr = init(null, { width: 480, height: 360, devicePixelRatio: dpr });

zr.add(new Text({
  style: {
    text: 'wasm-zrender 文本',
    x: 24,
    y: 48,
    fill: '#333',
    fontSize: 18,
  },
}));

const rgba = zr.refresh();
const ctx = canvas.getContext('2d');
ctx.putImageData(new ImageData(new Uint8ClampedArray(rgba), zr.width(), zr.height()), 0, 0);

site 薄壳辅助

文档站提供 site/src/zrender/fonts.js： loadFontFromUrl(url)、ensureDefaultFont()、registerFontBytes(bytes)。 默认字体文件：site/public/fonts/NotoSansSC-Regular.ttf。 text 示例 演示完整流程。

Wasmer / 原生 Rust 宿主

use rust_zrender::{register_font, RegisterFontOptions, ZRenderer};

register_font(font_bytes, RegisterFontOptions {
    family_name: Some("Noto Sans SC".into()),
    sans_serif: Some(vec!["Noto Sans SC".into()]),
})?;

let mut zr = ZRenderer::new(480, 360)?;

wasm32 目标不加载系统字体；原生目标默认仍加载系统字体，并与已注册字体合并。

ZRender 实例方法

  • add(el) / remove(el) — 根节点增删
  • refresh() / flush() — 同步返回 RGBA 字节数组
  • resize(opts) — 调整尺寸（不重载预设场景）
  • findHover(x, y) — 返回 { target, topTarget }

已实现图元

Group、Rect、Circle、Line、Polygon、Polyline、Sector、Text

Path 子类支持 useState(state)、setStateStyle(state, style)。

Stub 导出（构造时抛错）

Arc、BezierCurve、Image、Ellipse、 LinearGradient、RadialGradient、Pattern 等其余 export.ts 类型均已导出，便于后续补实现。

与官方 zrender 的差异

  • 仅 canvas 离屏模式，无 SVG painter / animation 帧循环
  • refresh() 同步返回像素，需 JS 侧 putImageData 上屏
  • init(dom) 的 dom 参数忽略
  • Text 暂仅支持 zr.add(text) 根节点（不支持 group.add(text)）
  • 文本须宿主注册字体，见上文「字体加载」；无浏览器系统字体回退
  • 无 on('click') 事件总线，命中检测需自行绑定 DOM 事件

架构关系

wasm-zrender 与 wasm-echarts 为 sibling，均只依赖 rust-zrender。 echarts 图表管线不经过 wasm-zrender，而是内部直接使用 ZRenderer。
