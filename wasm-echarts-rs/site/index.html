wasm-echarts

WebAssembly 图表渲染

用 Rust 重写 zrender 渲染内核，并在浏览器侧提供 wasm-zrender 与 wasm-echarts 两个 WASM 产物。

项目缘由

官方 ECharts 的 canvas 渲染依赖 JavaScript 与 zrender。本项目将渲染层迁移到 Rust， 通过 wasm-bindgen 暴露给浏览器，目标包括：

  • 性能：几何计算、绘制指令生成在 WASM 中执行，减少 JS 主线程开销
  • 可测试性：底层 rust-zrender 为纯 Rust crate，可用 cargo test 做像素级回归
  • 分层架构：rust-zrender（引擎）+ wasm-zrender（底层 API）+ wasm-echarts（图表 option 管线），职责清晰、可独立集成
  • 兼容路径：echarts 侧尽量对齐 init / setOption / 事件与交互语义，便于从官方 API 平滑迁移

已知限制

当前为 MVP 阶段，以下能力明确不在范围内或尚未实现：

  • 不支持 SVG 渲染模式；仅 canvas（WASM 出 RGBA → JS 写 ImageData）
  • 无动画中间帧 / morph；setOption 后为即时全量重绘
  • 无 Loading、DataView 等 DOM 组件；tooltip 以字符串为主
  • legend、dataZoom 滑块、gauge、polar 等组件/图表类型尚未完整
  • RichText / series label 引导线等待补全；与官方 ECharts 视觉/行为存在差异

详细 API 与差异说明见各产品文档页。
